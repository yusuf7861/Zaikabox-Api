# Swagger/OpenAPI Documentation for Zaikabox API

## Changes Made

1. Added springdoc-openapi dependency to pom.xml:
   ```xml
   <dependency>
       <groupId>org.springdoc</groupId>
       <artifactId>springdoc-openapi-starter-webmvc-ui</artifactId>
       <version>2.5.0</version>
   </dependency>
   ```

2. Added OpenAPI configuration in application.properties:
   ```properties
   # OpenAPI/Swagger Configuration
   springdoc.api-docs.path=/api-docs
   springdoc.swagger-ui.path=/swagger-ui.html
   springdoc.swagger-ui.operationsSorter=method
   springdoc.swagger-ui.tagsSorter=alpha
   springdoc.swagger-ui.tryItOutEnabled=true
   springdoc.swagger-ui.filter=true
   springdoc.swagger-ui.deepLinking=true
   springdoc.swagger-ui.displayRequestDuration=true
   springdoc.swagger-ui.syntaxHighlight.activated=true

   # API Information
   springdoc.info.title=Zaikabox API
   springdoc.info.description=API documentation for Zaikabox food delivery application
   springdoc.info.version=1.0.0
   springdoc.info.contact.name=Zaikabox Team
   springdoc.info.contact.email=info@zaikabox.com
   springdoc.info.license.name=API License
   ```

3. Updated SecurityConfig to allow access to Swagger UI endpoints:
   ```java
   .requestMatchers(
       "/api/v1/users/register", 
       "/api/v1/users/login", 
       "/api/v1/foods/**", 
       "/contact-us",
       "/swagger-ui.html",
       "/swagger-ui/**",
       "/api-docs/**",
       "/v3/api-docs/**"
   ).permitAll()
   ```

## Accessing Swagger UI

Once the application is running, you can access the Swagger UI at:
- http://localhost:8080/swagger-ui.html

The OpenAPI documentation in JSON format is available at:
- http://localhost:8080/api-docs
- http://localhost:8080/v3/api-docs

## Features

- Interactive API documentation
- Try out API endpoints directly from the UI
- Filter operations
- Syntax highlighting
- Request duration display
- Deep linking to specific operations

## Benefits

- Improved developer experience
- Self-documenting APIs
- Easier testing and debugging
- Better collaboration between frontend and backend teams
